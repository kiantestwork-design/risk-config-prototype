# 规则与策略集多对多改造设计

## 概述

将规则（Rule）从策略集（Activation）的附属实体改造为独立实体，规则与策略集之间建立多对多关系。规则作为可复用的「原子能力」，策略集作为「编排层」引用规则并设置优先级和权重。

## 数据模型

### Rule（规则 - 独立实体）

```
Rule
├── name: string              // 规则名称（唯一标识）
├── description: string       // 规则描述
├── conditionExpression        // 条件表达式（AND/OR 逻辑树）
├── initScore: number         // 初始分数
├── baseNum: number           // 基数
├── operator: RuleOperator    // 运算符 (ADD/SUB/MUL/DIV/NONE)
├── valueField: string        // 取值字段
├── max: number | null        // 分数上限
├── rate: number              // 倍率
├── actions: RuleAction[]     // 关联动作
├── status: FeatureStatus     // 运行状态 (ENABLED/DISABLED)
├── lifecycleState: FeatureLifecycle  // DRAFT → READY_FOR_RELEASE → PUBLISHED → ARCHIVED
├── createAt, updateAt, operator
```

**关键变更：**
- 删除 `activationName` 字段（不再从属于某个策略集）
- 删除 `eventPoint` 字段（一期去除接入点概念）
- 删除 `priority` 字段（优先级移到关联表，由策略集内决定）

### ActivationRule（关联表 - 新增）

```
ActivationRule
├── activationName: string    // 所属策略集
├── ruleName: string          // 引用的规则
├── priority: number          // 在该策略集内的优先级（数字越小越优先）
├── weight: number            // 在该策略集内的权重（默认 1.0）
```

**审核归属：** 关联表的变更（添加/移除规则、调整优先级/权重）纳入**策略集**的生命周期审核流程。

### Activation（策略集 - 不变）

保持现有结构不变。

## 审核流程

两条独立的审核线：

| 变更类型 | 审核归属 | 示例 |
|---------|---------|------|
| 规则的条件、打分、动作 | 规则自身的生命周期 | 修改 rule_high_amount 的条件表达式 |
| 关联关系（增删规则、优先级、权重） | 策略集的生命周期 | 给 activation_txn_risk 添加一条规则 |

## UI 设计

### 1. 规则列表页（改造）

**筛选条件：**
- 规则名称（关键词搜索）
- 生命周期状态（DRAFT / READY / PUBLISHED / ARCHIVED）
- 运行状态（ENABLED / DISABLED）

**列表列：**
- 规则名称
- 描述
- 生命周期状态
- 运行状态
- **引用数**（被多少个策略集引用，可点击展开查看列表）
- 操作（查看/编辑/删除）

**变更：**
- 去除「所属策略集」筛选
- 去除「事件点」筛选
- 新增「引用数」列

### 2. 规则编辑页（改造）

**变更：**
- 删除「所属策略集」下拉选择框
- 删除「事件点」字段
- 删除「优先级」字段（移到关联表）
- 新增**影响提示栏**：当规则被 ≥1 个策略集引用时，顶部显示橙色警告：
  > ⚠ 此规则被 N 个策略集引用，修改将影响：策略集A、策略集B...

  策略集名称可点击跳转。

### 3. 规则详情页（改造）

**变更：**
- 去除事件点、所属策略集信息
- 底部新增「引用方」区域，列出所有引用该规则的策略集（名称、事件点、状态）

### 4. 策略集详情页 — 关联规则面板（新增）

在策略集详情页中新增「关联规则」面板：

**表格列：**
- 拖拽手柄（⠿ 图标）
- 规则名称（可点击跳转到规则详情）
- 描述
- 优先级（可点击直接编辑）
- 权重（可点击直接编辑）
- 规则状态（来自规则本身的 status）
- 操作（移除）

**操作：**
- 「添加规则」按钮 → 打开规则选择弹框
- 拖拽排序 → 自动按 10 递增重算 priority
- 点击优先级/权重数字 → 内联编辑
- 移除 → 解除关联关系

### 5. 添加规则弹框

- 展示所有可用规则（不按事件点过滤）
- 支持关键词搜索（按名称和描述）
- 已关联的规则置灰不可重复选
- 支持多选（勾选框）
- 底部显示已选数量，确认/取消按钮

## 交互流程

### 策略集添加规则

1. 策略集详情页点击「添加规则」
2. 弹出规则选择弹框，展示所有规则
3. 已关联规则置灰，支持搜索和多选
4. 确认后，选中规则以默认优先级（末尾递增）加入关联列表
5. 策略集配置变更需走策略集发布审核

### 规则编辑影响提示

1. 进入规则编辑页
2. 查询引用方数量
3. 若 ≥1，顶部显示橙色提示栏，列出所有引用方策略集
4. 用户编辑后走规则自身的发布审核

### 拖拽排序

1. 拖拽规则行调整顺序
2. 拖拽结束自动重算 priority（10, 20, 30...）
3. 也可直接点击 priority 数字手动输入
4. 手动输入后列表按数字自动重排

## 数据迁移

现有 mock 数据迁移策略：
1. 从现有 Rule 中提取 `activationName` 和 `priority`，生成 ActivationRule 记录
2. 从 Rule 中删除 `activationName`、`eventPoint`、`priority` 字段
3. ActivationRule 默认 weight 设为 1.0
